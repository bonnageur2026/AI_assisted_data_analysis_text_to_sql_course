# AI 보조 데이터 분석과 Text-to-SQL 질문 설계 교육 과정

> Python 코딩 없이 파일 기반 EDA, SQL 독해, 질문 설계, AI 결과 검증을 배우는 실무형 교육 과정

---

## 제목에 관한 정직한 범위 설정

이 과정은 수강생이 SQL이나 Python을 직접 작성하게 하는 과정이 아니다. 또한 운영 데이터베이스에 실제로 쿼리를 실행하는 본격적인 Text-to-SQL 파이프라인 구축 과정도 아니다.

1차적으로 다루는 것은 AI 도구에 CSV/엑셀 파일을 올려 수행하는 **대화형 분석**, 즉 **파일 기반 EDA(Exploratory Data Analysis)**이다. Text-to-SQL은 **좋은 질문을 SQL이 풀 수 있는 형태로 설계하고, 생성된 쿼리를 읽고 검증하는 능력**에 초점을 둔다.

실제 DB 연동, 접근 권한, 로그 관리, API key 관리, 배포, 운영 환경 구축은 심화 과정의 범위다. 이 경계를 흐리지 않는 것이 과정 신뢰도의 핵심이다.

이 과정은 AI에게 분석을 맡기는 법만 배우는 과정이 아니다. 더 정확히 말하면, **AI가 수행한 분석을 사람이 질문하고, 검증하고, 해석하고, 실제 의사결정으로 연결하는 법을 배우는 과정**이다.
